# Entrevista — Analista financiero

> Notas crudas de la conversación. Entrevista simulada y anonimizada para identificar criterios financieros, riesgos y necesidades de explicación del producto.

**Entrevistador:** ¿Qué información debería procesar una aplicación de apoyo a decisiones de inversión?

**A. (analista financiero):** Mire, como mínimo debería revisar precios históricos, precios actuales, volumen, volatilidad e indicadores técnicos. También sería bueno que considere variables macroeconómicas, porque el mercado no se mueve solo por gráficos.

**Entrevistador:** ¿Cuál considera que es el dolor más grande en este tipo de productos?

**A.:** El mayor problema es que muchas aplicaciones entregan recomendaciones como si fueran verdades absolutas. Eso es delicado, porque una inversión siempre tiene riesgo. El usuario debe entender que no hay garantía de ganancia.

**Entrevistador:** ¿Qué debería mostrar el sistema junto con cada recomendación?

**A.:** Debería mostrar el nivel de riesgo, la explicación de la recomendación, los datos usados y un posible escenario de pérdida. No basta con decir “conviene invertir”; hay que justificarlo.

**Entrevistador:** ¿Qué riesgos legales o éticos observa?

**A.:** Que el usuario interprete la recomendación como asesoría financiera personalizada y garantizada. Por eso se deben incluir advertencias claras, términos de uso y transparencia sobre los cálculos.

**Entrevistador:** ¿Qué considera indispensable para generar confianza?

**A.:** Explicar el porqué de cada resultado. Si el sistema recomienda algo, debe mostrar qué variables influyeron: tendencia, volatilidad, perfil de riesgo, horizonte de inversión y comportamiento histórico.
